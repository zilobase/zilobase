# Governance

Notelab is currently maintained by the project owner and invited maintainers.

## Decision Making

Maintainers make final decisions on roadmap, architecture, releases, and whether to merge pull requests. Decisions are based on project direction, maintainability, user impact, security, and review bandwidth.

For substantial changes, open an issue or discussion before implementation. This is especially important for:

- Database schema changes
- Public API changes
- Authentication or authorization changes
- New deployment targets
- Large UI or editor behavior changes
- New third-party integrations

## Maintainer Responsibilities

Maintainers are responsible for:

- Reviewing issues and pull requests
- Keeping the project direction coherent
- Protecting user security and privacy
- Enforcing the Code of Conduct
- Publishing releases and security fixes

## Contributors

Contributors can participate through issues, pull requests, documentation, testing, and design feedback. Contributions are accepted under the repository MIT License.

## Changes to Governance

This governance model may evolve as the project and contributor base grow.
